# Hidden VIP Finder

법인 고객의 거래 데이터를 이용해 `일반 → 우수 → 최우수` 등급을 예측하고, 아직 최우수로 분류되지는 않았지만 최우수 고객과 비슷한 패턴을 보이는 **Hidden VIP**를 찾는 머신러닝 프로젝트입니다.

단순히 등급을 맞히는 모델에서 끝내지 않고, SHAP 해석을 통해 업종별 성장 레버를 찾고 고객별 Next Best Action까지 연결하는 것을 목표로 했습니다.

## 문제 정의

은행의 법인 고객 관리는 거래 규모, 등급, 전담 여부 같은 정적 기준에 의존하기 쉽습니다. 이 프로젝트에서는 현재 등급만 보는 대신, 고객의 거래 패턴이 최우수 고객과 얼마나 가까운지 확률로 추정했습니다.

분석에서 다룬 질문은 네 가지입니다.

- 현재는 최우수가 아니지만 최우수 가능성이 높은 고객은 누구인가?
- 업종별로 최우수 고객을 설명하는 핵심 변수는 무엇인가?
- 고객별로 어떤 상품/거래 행동을 제안하면 승격 가능성이 커지는가?
- 전담 관리가 비어 있는 Hidden VIP 중 누구를 먼저 배정해야 하는가?

---

## 분석 흐름

```text
EDA
  ↓
누수 변수 제거 및 전처리
  ↓
PyCaret AutoML 모델 비교
  ↓
최종 등급 예측 모델 학습
  ↓
Hidden VIP 탐지
  ↓
SHAP 기반 업종별 해석
  ↓
성장 레버 및 NBA 시뮬레이션
  ↓
전담 배정 우선순위 산출
```

### 1. EDA

기준년월별 고객 수, 업종 분포, 등급 분포, 결측률, 0값 비율을 먼저 확인했습니다. 법인 금융 데이터 특성상 수익증권, 외환, 퇴직연금, 신탁 등은 0값 비율이 높았고, 이 값 자체가 상품 보유 여부를 나타내는 의미 있는 신호로 해석될 수 있었습니다.

<img width="2961" height="2647" alt="eda_step1_overview" src="https://github.com/user-attachments/assets/00711cce-f90b-410e-aeff-394f12049070" />

### 2. 전처리

모델 성능보다 먼저 데이터 누수 방지에 신경 썼습니다.

- `전담고객여부`는 메인 등급 예측에서 제외  
  높은 등급이라서 전담이 붙었는지, 전담이라서 등급이 높아졌는지 방향이 섞여 있기 때문입니다.
- `사업장_시군구`, `기준년월`, `업종_중분류` 등 식별성 또는 과적합 위험이 큰 변수 제거
- Train / Validation / Test 분리 후 Train 기준으로만 전처리 fit
- 업종 내 Z-score, 결측 대치, One-Hot Encoding, 피처 선택을 모두 Train 기준으로 수행
- Isolation Forest 이상치 제거도 Train에만 적용

최종 모델 입력 피처는 총 23개로 구성했습니다.

### 3. 모델링

PyCaret으로 여러 트리 기반 모델을 비교한 뒤 최종 후보를 선정했습니다. 모델 평가는 Accuracy만 보지 않고, 다중분류 상황에 맞춰 ROC-AUC, PR-AUC, F1 macro를 함께 확인했습니다.

주요 성능은 다음 수준으로 확인됐습니다.

| 지표 | 결과 |
| --- | ---: |
| ROC-AUC | 약 0.95 |
| PR-AUC | 약 0.91 |
| F1 macro | 약 0.84 |

## 주요 결과

### Hidden VIP 발굴

비최우수 고객 중 `P(최우수)`가 임계값 이상인 고객을 Hidden VIP로 정의했습니다. 업종별로 Hidden VIP 비율과 평균 최우수 확률을 비교해, 단순 고객 수가 많은 업종과 실제 잠재 고객 비중이 높은 업종을 구분했습니다.

## NBA 설계

고객별 추천 액션은 5개 그룹으로 나눴습니다.

| 액션 그룹 | 주요 피처 | 해석 |
| --- | --- | --- |
| 수신 확대 | 요구불예금잔액, 요구불입금/출금금액, 요구불회전율 | 거래 기반 수신 확대 |
| 자산관리 유치 | 신탁잔액, 퇴직연금잔액 | 여유자금 및 장기성 자금 유치 |
| 여신 확대/구조화 | 여신한도, 운전자금, 시설자금 | 대출 한도 및 자금 용도 재설계 |
| 디지털 채널 전환 | 디지털채널비율, 인터넷뱅킹거래금액 | 채널 사용 패턴 개선 |
| 카드/자동화 활성화 | 신용카드사용금액, 자동이체금액/건수 | 운영성 거래 확대 |

각 액션 그룹에 포함된 피처를 업종별 최우수 평균값으로 대체했을 때의 `ΔP(최우수)`를 계산하고, 가장 큰 상승폭을 보이는 액션을 고객별 추천 액션으로 정했습니다.

## 전담 배정 우선순위

Hidden VIP를 전담 고객 여부에 따라 두 그룹으로 나눴습니다.

| 세그먼트 | 정의 | 대응 방향 |
| --- | --- | --- |
| 관리중 미전환 | Hidden VIP + 전담 고객 | 기존 담당 RM에게 NBA 액션 전달 |
| 공략공백 | Hidden VIP + 비전담 고객 | 전담 배정 후보로 우선 검토 |

이후 별도의 이진 분류 모델로 `전담고객여부`를 예측해, 공략공백 고객 중 현재 은행의 전담 배정 패턴에 가까운 고객을 우선순위화했습니다.

이 모델은 "전담 배정을 하면 등급이 오른다"는 인과 모델이 아니라, 현재 전담 배정 패턴을 학습해 영업 검토 대상을 좁히는 보조 모델입니다.

---
### 최종 산출물 요약

최종적으로 Hidden VIP를 전담 관리 상태와 액션 추천 가능 여부까지 함께 정리했습니다.

| 구분 | 고객 수 |
| --- | ---: |
| 전체 Hidden VIP | 3,787명 |
| 관리중 미전환 | 3,408명 |
| 공략공백 | 379명 |
| 즉시 전담 배정 추천 | 104명 |

여기서 **즉시 전담 배정 추천 104명**은 `공략공백` 고객 중에서도 전담 필요도 예측이 높고, 추천 액션의 기대 효과(`Delta`)가 유효하게 나온 고객입니다.

즉, 단순히 "최우수 가능성이 높은 고객"이 아니라 **아직 전담 관리가 비어 있고, 액션을 제안했을 때 등급 상승 가능성도 함께 기대되는 고객군**으로 볼 수 있습니다.

## 한계와 개선 방향

- **액션 추천 효과 검증**  
  NBA는 "이 피처가 최우수 고객 평균 수준으로 바뀌면 확률이 얼마나 움직이는가"를 본 시뮬레이션입니다. 실제 영업 캠페인에서는 고객 반응률, 실행 가능성, 상품 적합성까지 같이 봐야 하므로 A/B 테스트나 파일럿 캠페인 결과로 한 번 더 검증해야 합니다.

- **시간 흐름 반영**  
  현재 모델은 36개월 데이터를 사용했지만, 고객별 월별 변화 패턴을 시계열 모델처럼 직접 학습한 구조는 아닙니다. 예를 들어 최근 3개월 수신 증가율, 여신 잔액 변화 추세, 거래 빈도 하락 같은 흐름을 더 정교하게 넣으면 조기 탐지 성능을 높일 수 있을 것 같습니다.

- **업종별 기준 세분화**  
  제조업, 부동산업, 도소매업처럼 거래 구조가 다른 업종을 하나의 모델에서 함께 다뤘습니다. SHAP으로 업종별 차이를 해석하긴 했지만, 데이터가 충분한 업종은 별도 모델이나 업종별 임계값을 두는 방식도 비교해볼 만합니다.

- **추천 액션의 현실성 점검**  
  모델상으로는 특정 피처를 올리는 것이 좋아 보여도, 실제로는 고객 상황상 실행하기 어려운 액션이 있을 수 있습니다. 향후에는 추천 액션마다 실행 난이도, 예상 수익, RM 투입 비용을 함께 반영해 우선순위 점수를 더 현실적으로 만들 수 있습니다.

- **전담 배정 모델의 해석 주의**  
  전담 배정 필요도 모델은 "현재 은행이 어떤 고객에게 전담을 붙이는 경향이 있는지"를 학습한 보조 모델입니다. 따라서 이 결과를 그대로 정답처럼 쓰기보다는, 공략공백 고객을 빠르게 좁히는 필터로 사용하는 편이 더 적절합니다.

## 한 줄 정리

이 프로젝트는 법인 고객 등급 예측을 출발점으로 삼아, Hidden VIP 발굴, 업종별 성장 레버 해석, 고객별 액션 추천, 전담 배정 우선순위까지 이어지는 실행형 분석 파이프라인입니다.
